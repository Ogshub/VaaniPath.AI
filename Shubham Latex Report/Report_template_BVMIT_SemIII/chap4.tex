\chapter{System Design}

\section{Entity Relationship (ER) Diagram}

The data model for VaaniPath AI centers on document processing entities rather than complex user management.
\begin{enumerate}
    \item \textbf{User Session:} The active interaction period.
    \item \textbf{Source/Output Document:} The raw input and the final generated PDF.
    \item \textbf{Page Object:} The atomic unit of processing containing text and images.
    \item \textbf{Translation Job:} Stores settings (Language, Mode) and process status.
\end{enumerate}

\section{Activity Diagram}

The system follows a linear execution flow:
\begin{enumerate}
    \item \textbf{Upload \& Config:} User uploads file and selects translation mode.
    \item \textbf{Analysis:} System checks if the file is digital or scanned.
    \item \textbf{Extraction:} Uses direct extraction for digital PDFs, or OCR for scanned ones.
    \item \textbf{Translation \& Reconstruction:} Translates text and overlays it on the canvas.
    \item \textbf{Download:} Generates final PDF for user.
\end{enumerate}
